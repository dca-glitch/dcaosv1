# Registry Pattern

## Purpose

The registry pattern lets the platform discover modules and capabilities from metadata.

## Registry Entry

A registry entry may include:

- key
- name
- description
- category
- route
- navigation item
- dashboard cards later
- permission keys later

## Static Phase

A registry may start as static TypeScript data.

## Runtime Phase

Later, registry data may combine static definitions with tenant access rules.

## Rule

Use registry metadata to reduce duplicated module wiring.
