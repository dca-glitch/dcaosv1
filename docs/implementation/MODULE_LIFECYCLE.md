# Module Lifecycle

## Planned

The module is documented but not implemented.

## Static

The module has metadata, contracts, and placeholder UI or API behavior.

## Integrated

The module is connected to API and database access patterns.

## Tenant-Aware

The module respects tenant context.

## Access-Controlled

The module respects roles and permission checks.

## Production-Ready

The module has validation, audit events, operational notes, and tested behavior.

## Rule

Do not label a module production-ready before tenant and access behavior are complete.
