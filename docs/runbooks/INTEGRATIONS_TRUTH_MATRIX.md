## Summary rules confirmed by code inspection

- Every integration follows the same **disabled → missing_config → configured_shape_ok** three-state pattern, and every readiness category explicitly self-labels as live-calls-deferred (`liveCallsDeferred: true` / equivalents) at the type level in `external-integrations-readiness.service.ts`.
- No integration in this codebase currently claims live-proven status anywhere in code or docs that this audit could find — all live proof requirements are consistently deferred pending separate owner approval, consistent with `G53_PRODUCTION_SAFETY_PLAN.md` §4.2.
- Two gaps for a future small block (not blockers, just completeness): (1) image generation has scaffold but no readiness category yet — add when provider is wired per `IMAGE_GENERATION_PROOF.md`; (2) email/Resend readiness is not yet exposed via the unified `GET /api/v1/integrations/readiness` endpoint — it is only visible via the email outbox smoke.
- **SEC-B1 (2026-07-09):** Legacy `/api/v1/briefs` cross-tenant IDOR fixed locally; regression test `briefs-tenant-boundary.integration.test.ts`. Not a launch integration row but a production safety prerequisite.
