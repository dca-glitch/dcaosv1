# DCA OS v1 / DCA OS Lite — Approved Dark Nebula Product UI Direction

Status: Approved

This document records the approved product UI direction for DCA OS v1 / DCA OS Lite.

## Approval

The generated Dark Nebula mockups are the target product design system and product experience. They are not a simple CSS skin.

Implementation should follow this direction across the whole frontend:

- layout structure
- navigation/sidebar behavior
- dashboard composition
- cards and panels
- module pages
- status badges
- forms and modals
- spacing and typography
- action hierarchy
- empty/loading/error states
- overall visual polish

## Product direction

DCA OS v1 / DCA OS Lite should evolve toward a polished dark SaaS interface with:

- deep navy / near-black backgrounds
- subtle nebula gradients
- blue-violet, cyan, and soft purple accents
- glass panels and cards
- soft translucent borders
- strong readable typography
- spacious cards and page sections
- clear operational workflows
- consistent dark-theme badges
- accessible contrast
- simple card/modal-first UX

## Implementation boundary

The approved UI direction must preserve:

- existing backend logic
- existing routes
- existing authentication behavior
- existing Turnstile behavior
- existing API contracts
- existing database schema
- production safety rules

No backend, database, auth, Turnstile, VPS, or deployment changes are implied by this approval.
